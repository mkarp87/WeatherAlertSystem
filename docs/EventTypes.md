# NWS v1.2 Event Names

Weather Alert System filters alerts using the NWS alert `event` string. Put these values under `nws.included_events` to make an allow-list. Leave `included_events: []` to allow all events except those listed in `excluded_events`.

These names are included for compatibility with the SkyWarnPlus documentation, which states that SkyWarnPlus supports all 128 alert types in the NWS v1.2 API.

## Config example

```yaml
nws:
  included_events:
    - "Tornado Warning"
    - "Severe Thunderstorm Warning"
    - "Flash Flood Warning"
  excluded_events:
    - "Test Message"
```

## Notes

- Matching is exact against the NWS `event` field.
- The Weather Alert System default is `included_events: []`, which means all NWS event names are allowed unless explicitly excluded.
- Production systems usually keep `excluded_events: ["Test Message"]` or similar to avoid official test traffic.
